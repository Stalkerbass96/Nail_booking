# PRD V3（LINE First 2.0）

- 版本：V3
- 对应需求冻结：V2.0 LINE First
- 更新日期：2026-03-08

## 1. 产品目标

构建一个以 LINE 为主入口的美甲预约系统：
- 顾客通过线下扫码加好友进入系统。
- 系统自动沉淀顾客资料，并通过 LINE 驱动预约与确认。
- 前台首页升级为图墙，顾客按图选款预约。
- 店长在后台维护图墙、预约和顾客，并通过 LINE 与顾客确认细节。

## 2. 2.0 成功标准

1. 顾客加好友后自动建档成功。
2. LINE 首次自动推送首页链接成功。
3. 首页图墙可正常展示、筛选、点击进入预约。
4. 顾客按图墙项完成预约后，LINE 自动收到待确认通知与预约详情链接。
5. 店长后台确认预约后，顾客可收到 LINE 确认通知。
6. 店长可在后台管理图墙，并通过 LINE 与顾客进行文本沟通。

## 3. 角色与权限

### 3.1 顾客

- 通过 LINE 加好友进入系统
- 浏览图墙
- 按图墙项创建预约
- 查看预约详情

### 3.2 店长

- 登录后台
- 管理图墙
- 管理套餐、分类、加项
- 管理预约状态
- 查看潜在客户/正式顾客
- 使用 LINE 与顾客 1 对 1 文本沟通

## 4. 产品范围

### 4.1 顾客侧（2.0 首批）

- 加好友后自动创客
- LINE 首次自动推送首页链接
- 图墙首页
- 图墙分类筛选
- 通过图墙项进入预约
- 预约提交
- 预约详情公开页
- LINE 待确认通知
- LINE 已确认通知

### 4.2 店长侧（2.0 首批）

- 图墙管理
- 预约管理
- 顾客资料维护
- LINE 会话管理（文本消息）

- 排班/封锁继续管理

### 4.3 不在 2.0 首批

- 图墙多图详情
- LINE 图片消息

- 多店模型

## 5. 核心流程

### 5.1 加好友到预约

1. 顾客扫码添加店铺 LINE 官方账号。
2. Webhook 收到 `follow` 事件。
3. 系统以 `LINE userId` 创建顾客档案，状态为 `lead`。
4. 系统仅首次向该顾客推送首页链接。
5. 顾客打开 `/` 图墙首页。
6. 顾客按分类筛选并点击某图墙项。
7. 系统带着该图墙项绑定的套餐进入预约页。
8. 顾客选择时间并填写备注。
9. 系统创建预约：
   - 状态 `pending`
   - 占用档期
   - 顾客转为 `active`
10. LINE 推送待确认消息与预约详情链接。

### 5.2 店长确认与会话

1. 店长在后台看到待确认预约。
2. 若有需要，可先在后台通过 LINE 文本消息确认细节。
3. 店长确认预约。
4. 系统更新预约状态为 `confirmed`。
5. LINE 推送预约确认消息。

## 6. 信息架构

### 6.1 顾客侧页面

- `/`：图墙首页
- `/booking?showcaseItemId=...`：预约页
- `/booking/success/{bookingNo}`：预约成功页
- `/booking/{bookingNo}` 或等价公开详情页：预约详情页

### 6.2 店长侧页面

- `/admin`
- `/admin/showcase`：图墙管理
- `/admin/appointments`
- `/admin/customers`
- `/admin/line`
- 其余已有管理页继续保留

## 7. 业务规则

1. 顾客主身份是 `LINE userId`。
2. 顾客加好友即自动建档。
3. 顾客首次预约提交即从 `lead` 转成 `active`。
4. 邮箱不是必填项，也不是预约查询条件。
5. 图墙项只能绑定一个套餐，但多个图墙项可关联同一套餐。
6. 顾客从图墙项进入后不能改套餐。
7. 预约状态维持现有四态模型。
8. `pending` 继续占用档期。
9. LINE 首次加好友时只推一次首页链接。
10. 店长与顾客会话首期只支持文本消息。

## 8. 后台管理重点

### 8.1 图墙管理

- 新建/编辑/删除图墙项
- 上传封面图
- 配置标题、说明、分类、价格文案
- 绑定套餐
- 控制上下架与排序

### 8.2 顾客管理

- 区分潜在客户与正式顾客
- 默认显示 LINE 昵称
- 支持后台修正姓名等信息
- 关联历史预约和 LINE 会话

### 8.3 预约管理

- 查看预约来源图墙项
- 确认/取消/完成预约
- 结合 LINE 消息确认需求

## 9. 验收标准

- `follow` 事件后可自动创建顾客
- 后台可见潜在客户
- 图墙首页可展示并筛选
- 图墙项点击后可进入固定套餐预约流程
- 提交预约后档期被占用
- 提交预约后 LINE 消息包含预约号和详情链接
- 店长确认预约后 LINE 可收到确认消息
- 后台可维护图墙与顾客资料
- 后台可与顾客进行 1 对 1 文本会话
