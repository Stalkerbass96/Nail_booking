# PRD V2（可开发版）

- 版本：V2
- 对应需求冻结：V1.1
- 更新日期：2026-03-06

## 1. 项目目标

构建单店美甲预约系统，实现从“顾客预约”到“店长管理”的最小闭环：

- 前台可浏览服务并提交预约。
- 后台可处理预约状态并管理服务目录、客户与积分。
- 核心流程支持中日双语。

## 2. 角色与权限

- 顾客（前台）：浏览、预约、查询预约。
- 店长/美甲师（后台）：登录后管理预约与业务数据。

## 3. MVP 范围

### 3.1 前台

- 分类与套餐展示（含加项信息）。
- 预约创建（待确认状态）。
- 可用时段计算与冲突拦截。
- 预约成功页（展示预约编号）。
- 预约查询（邮箱 + 预约号）。

### 3.2 后台

- 登录鉴权。
- 预约流转（确认、取消、完成）。
- 分类管理（新建、编辑、批量启停、排序调整）。
- 套餐管理（新建、编辑、关联加项）。
- 加项管理（新建、编辑）。
- 客户信息与历史查询。
- 积分流水查看与扣减。

### 3.3 系统任务

- 待确认预约超时自动取消并释放档期。

### 3.4 不在 MVP

- 在线支付。
- 邮件通知。
- LINE 集成。
- 后台代客预约。

## 5. 业务规则

1. 单技师模型，同时间不可重复预约。
2. 预约创建后状态为 `pending`，并占用档期。
3. 状态流转：`pending -> confirmed -> completed`，或 `pending/confirmed -> canceled`。
4. `pending` 超时自动取消并释放档期。
5. 取消限制按预约开始前 X 小时计算（当前默认 6 小时）。
6. 总时长 = 套餐时长 + 加项时长总和。
7. 预约时间粒度固定 30 分钟。
8. 积分累计：`floor(实际支付日元 / 100)`。
9. 积分抵扣：`1 point = 100 日元`。

## 6. 验收标准（MVP）

- 可成功创建预约并生成唯一预约编号。
- 冲突时段预约被拦截。
- 后台可确认/取消/完成预约。
- 待确认预约可自动取消。
- 完成预约可累计积分，并可记录积分扣减流水。
- 顾客可通过邮箱 + 预约号查询状态。
- 前后台核心流程支持中日双语。
