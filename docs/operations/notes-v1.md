# 备注与已知限制 V1

- 更新日期：2026-03-06

## 1. 设计备注

- 当前为单店、单技师模型，冲突判定按单资源实现。
- 预约时间粒度固定为 30 分钟。
- `pending` 状态会占用档期，避免重复预约。
- 规则参数存储于 `system_settings`，默认值由 `seed` 写入。
- 已提供 Docker 一体部署：`postgres + app + auto-cancel-worker`。

## 2. 已知限制（当前版本）

1. 系统设置管理页未落地（暂通过数据库维护）。
2. 邮件通知未实现。
3. LINE 集成未实现。
4. 在线支付未实现。
5. 自动化测试覆盖不足（目前以手工验证 + build 为主）。

## 3. 运行维护备注

- 自动取消任务属于业务关键路径，必须保持 worker 在线。
- 生产发布后需检查管理员账号密码是否已替换默认值。
- 生产环境应启用 HTTPS，并保护好 `.env` / `.env.deploy`。
- 建议每次发布后验证：
  1. `/admin/login` 可访问
  2. 登录成功
  3. 预约创建与查询可用
  4. worker 日志正常输出

## 4. 建议的下一阶段工作

- P0：后台系统设置页（营业时间、取消窗口、自动取消窗口、积分比例）。
- P1：通知能力（邮件）与模板体系。
- P1：关键 API 的单元测试与集成测试。
- P2：统一 i18n 资源管理（减少硬编码文案分散）。
