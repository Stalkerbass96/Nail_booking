# 备注与已知限制 V1

- 更新日期：2026-03-07

## 1. 设计备注

- 当前为单店、单技师模型，冲突判定按单资源实现。
- 预约时间粒度默认 30 分钟（可在系统设置中调整）。
- `pending` 状态会占用档期，避免重复预约。
- 规则参数存储于 `system_settings`，并支持后台可视化维护。
- 已提供 Docker 一体部署：`postgres + app + auto-cancel-worker`。

## 2. 当前已落地（相对首版新增）

1. 系统设置管理页已落地（`/admin/settings`）。
2. 安全加固已完成：
   - `ADMIN_AUTH_SECRET` 必填（无默认回退）。
   - `CRON_SECRET` 必填（系统任务接口强制鉴权）。
3. 预约并发占位已加防护（事务内锁 + 冲突校验）。
4. E2E 测试链路已跑通，最新结果 `5/5 PASS`。

## 3. 已知限制（当前版本）

1. 邮件通知未实现。
2. LINE 集成未实现。
3. 在线支付未实现。
4. i18n 文案仍有部分分散在组件内（尚未完全资源化）。

## 4. 运行维护备注

- 自动取消任务属于业务关键路径，必须保持 worker 在线。
- 生产发布后需检查管理员账号密码是否已替换默认值。
- 生产环境应启用 HTTPS，并保护好 `.env` / `.env.deploy`。
- 建议每次发布后验证：
  1. `/admin/login` 可访问
  2. 登录成功
  3. 预约创建与查询可用
  4. worker 日志正常输出
  5. `npm run test:e2e` 或等效测试通过

## 5. 建议的下一阶段工作

- P1：通知能力（邮件）与模板体系。
- P1：在线支付能力（下单与回调闭环）。
- P1：统一 i18n 资源管理（减少硬编码文案分散）。
